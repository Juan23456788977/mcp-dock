# Contributing to this project

First off, thank you for considering contributing! It's people like you that make the open-source community such an amazing place to learn, inspire, and create.

## Development Workflow
1. Fork the repo and create your branch from `main`.
2. Run `npm install` to install dependencies.
3. Make your architectural changes ensuring you follow our State Management (Zustand) and Server components constraints.
4. Ensure all TypeScript interfaces are strictly typed.

## Pull Request Process
- Ensure any install or build dependencies are removed before the end of the layer when doing a build.
- Update the `README.md` with details of changes to the interface, including new environment variables if necessary.
- You may merge the Pull Request in once you have the sign-off of the core maintainers.
